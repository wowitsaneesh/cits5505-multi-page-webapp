Introduction

Websites may look like one complete product when we open them in a browser, but they are usually built using three different technologies working together: HTML, CSS, and JavaScript. Each one has its own role. If you understand the role of each technology, it becomes much easier to understand how webpages are built and why different parts of a website behave the way they do.

HTML is used to create the structure of a webpage. It tells the browser what the content is. For example, HTML can tell the browser that one line is a heading, another part is a paragraph, another item is a link, and another item is an image. CSS is used to control presentation. It changes how the structured HTML content looks, including colours, spacing, fonts, borders, and layout. JavaScript adds behaviour and interactivity. It allows the webpage to react when the user clicks, types, submits a form, or triggers some other event.

You can think of these three technologies in a simple way. HTML is the structure, CSS is the style, and JavaScript is the behaviour. A useful comparison is a house: HTML is like the walls, doors, and rooms; CSS is like the paint, decorations, and furniture arrangement; JavaScript is like the electrical systems and moving parts that make things respond and work dynamically.

In this project, all three are used together. HTML creates the tutorial page, quiz page, reflection page, and CV page. CSS gives the website its visual identity and readable layout. JavaScript powers the quiz, checks answers, shows feedback, stores attempts, and makes the page interactive. By the end of this tutorial, you should understand the main ideas clearly enough to answer the quiz.

HTML Fundamentals

HTML stands for Hyper Text Markup Language. It is the standard language used to structure content on a webpage. The word “markup” means that HTML marks different parts of content so the browser knows what they represent. HTML does not mainly control colours, animations, or advanced visual appearance. Instead, its main job is to organise the content into meaningful parts.

For example, when you see a large page title at the top of a website, that is usually written using a heading tag in HTML. When you see a normal block of writing, that is usually written using a paragraph tag. When you click text that takes you to another page, that is created using a hyperlink tag. When you see a picture on the screen, that is placed into the page using an image tag.

This is important because browsers do not just display text randomly. They interpret HTML tags and use them to decide how content should be organised. Search engines, accessibility tools, and screen readers also depend on this structure. So good HTML makes a page not only visible, but also understandable and usable.

Common HTML Tags

  • <h1> to <h6> are used for headings
  • <p> is used for paragraphs
  • <a> is used for hyperlinks
  • <img> is used for images
  • <ul> and <ol> are used for lists

Heading tags are used for titles and section names. <h1> is usually the main heading of the page and is normally the most important and largest heading. <h2> is commonly used for section headings, and <h3> can be used for smaller subheadings inside a section. Paragraph tags are used for normal written explanations. Hyperlink tags allow users to move to other pages or websites. Image tags display pictures. List tags help organise related points in a clear way.

HTML Example 1

<h1>Welcome</h1>
<p>This is a paragraph on the page.</p>

In this example, the first line creates a main heading. This tells the browser that “Welcome” is not just normal text, but an important title. The second line creates a paragraph. This tells the browser that the sentence is part of the main body content. Even though the example is small, it shows the basic idea of HTML: it gives structure and meaning to text.

HTML Example 2

<a href="quiz.html">Go to the quiz</a>
<img src="images/dawg.jpg" alt="Example image">

The first line creates a hyperlink. The part inside href tells the browser where the link should go when clicked. In this case, it points to the quiz page. The second line inserts an image. The src attribute tells the browser where to find the image file, and the alt attribute gives alternative text. This alternative text is useful if the image cannot load, and it is also important for accessibility tools such as screen readers.

So, when you are asked which HTML tag is used for a paragraph, the answer is <p>. When you are asked which tag inserts an image, the answer is <img>. When you are asked which element creates a hyperlink, the answer is <a>. These are basic but essential HTML ideas.

CSS Fundamentals

CSS stands for Cascading Style Sheets. It is used to control how a webpage looks. While HTML gives the page structure, CSS changes the presentation of that structure. This includes things like text colour, background colour, spacing, font size, borders, width, alignment, and layout.

Without CSS, a webpage can still work, but it often looks very plain. CSS is what helps turn plain structure into a cleaner, more attractive, and more readable design. For example, CSS can make a header blue, add padding around a section, change a button colour, round the corners of a box, or make text easier to read.

CSS works by using selectors. A selector tells the browser which HTML elements should receive a particular style rule. After the selector, CSS uses braces and property-value pairs. A property is the thing you want to change, such as color or font-size. The value is the setting you want to apply, such as blue or 18px.

What CSS Can Control

  1. Text colour and background colour
  2. Spacing, including padding and margins
  3. Font size, style, and emphasis
  4. Border styling and box appearance
  5. Layout and alignment of content

Text colour changes the colour of written content. Background colour changes the colour behind an element. Padding adds space inside an element, between its border and content. Margin adds space outside an element, separating it from nearby elements. Font size changes how large the text appears. Borders create visible edges around elements. Layout rules help organise where items appear on the page.

CSS Example 1

p {
    color: blue;
    font-size: 18px;
}

This example selects all paragraph elements using the selector p. It then changes the text colour to blue and makes the paragraph text larger by setting the font size to 18 pixels. This shows that CSS can apply the same style to every matching element at once.

CSS Example 2

#main-header {
    background-color: blue;
}

.container {
    padding: 20px;
}

The selector #main-header is an id selector. The # symbol means that CSS is targeting one specific element whose id is main-header. In this case, the rule changes its background colour to blue.

The selector .container is a class selector. The . symbol means CSS is targeting elements that use the class name container. In Bootstrap, .container is a very common class because it helps keep page content centred and spaced neatly across the screen.

So, if you are asked what the main purpose of CSS is, the answer is that CSS controls the style and appearance of a webpage. If you are asked which selector targets an element with id main-header, the answer is #main-header. If you are asked which Bootstrap class commonly keeps page content centred with responsive spacing, the answer is .container.

JavaScript Fundamentals

JavaScript is used to add behaviour and interactivity to webpages. Without JavaScript, a page is usually mostly static, which means it can display information but cannot do very much in response to the user. With JavaScript, the page can react when a user clicks a button, types into a field, submits a form, or interacts with the page in some other way.

JavaScript can also change content that is already on the page. For example, it can show popup messages, ask the user to type something, perform calculations, change text inside a section, save information in the browser, or load data from another file. In this project, JavaScript is what makes the quiz interactive.

This means JavaScript is not mainly used for structure or pure styling. Its main role is behaviour. It is what allows a webpage to do something instead of only displaying information.

Common JavaScript Uses

  • Showing popup messages with alert()
  • Asking for input with prompt()
  • Printing information with console.log()
  • Saving data using localStorage
  • Updating page content dynamically

The function alert() displays a popup message. The function prompt() asks the user to enter some input. The function console.log() prints information to the browser console, which is useful during testing and debugging. The feature localStorage saves small pieces of data in the browser so they can still be available later.

JavaScript Example 1

alert("Welcome to JavaScript!");

This line shows a popup message to the user. It is one of the simplest examples of JavaScript interactivity. Nothing on the page structure changes, but the browser responds by displaying a message box.

JavaScript Example 2

let userName = prompt("What is your name?");
localStorage.setItem("name", userName);

In the first line, prompt() asks the user to type something. In the second line, localStorage saves that value in the browser. In this project, the same browser storage idea is used to save quiz attempts on the client side.

So, if you are asked what JavaScript is mainly used for, the answer is adding behaviour and interactivity. If you are asked which function can ask the user to type something, the answer is prompt(). If you are asked which browser feature can save quiz attempts on the client side, the answer is localStorage.
