Introduction

This page documents how AI tools were used during the development of this project and critically evaluates both their usefulness and their limitations. AI was used as a support tool for planning, drafting, debugging, refining, and reviewing parts of the website. However, it was not treated as an automatic source of correct answers. Every suggestion still had to be checked against the project brief, the marking rubric, lecture content, and browser testing.

The main benefit of AI was speed. It helped turn rough ideas into a starting point more quickly, suggested code structure, explained concepts in simpler language, and helped identify practical next steps. At the same time, the project showed that AI output can sound convincing even when it is too generic, slightly incorrect, incomplete, or misaligned with the exact requirements of the task.

AI Tool Used

ChatGPT was the main AI tool used during this project. It was used for planning page structure, improving written explanations, explaining HTML, CSS, and JavaScript concepts, reviewing progress against the project brief, and helping identify layout or logic problems during development.

Purpose of Using AI

AI was used mainly to support productivity and understanding. In the early stages of the project, it was useful for turning assignment requirements into smaller tasks that could be completed one at a time. During implementation, it helped draft page structure, suggest styling improvements, and provide starter JavaScript for interactive features. During later stages, it helped review whether the website still matched the assignment brief and rubric.

This was especially helpful because the project required several technologies to work together: HTML for structure, CSS for presentation, JavaScript for interactivity, AJAX for quiz loading, local storage for attempt history, and a public API for reward content. AI made it easier to move between those different parts of the project, but it did not remove the need to understand them.

Example Prompt Set 1: Tutorial Page Development

One major use of AI was in building and refining the tutorial page. The page had to explain HTML, CSS, and JavaScript clearly, include examples and code snippets, and also include interactive demonstrations.

Prompt example:
Generate a set of simple questions based on the concepts of basic HTML, CSS and JavaScript, and ideas for interactive demonstrations.

What the AI produced:
The AI generated a useful starting draft questions with sections for HTML, CSS, and JavaScript, along with ideas for a basic interactive element. Structurally, this was helpful because it gave a clear first idea of the website. However, the explanations were too brief and assumed too much prior knowledge. The draft also did not initially include enough detail for a beginner to feel fully guided through the concepts.

Corrections and changes made:
I expanded the explanations significantly so that a beginner could follow them from the ground up. I added more detailed examples, clarified the purpose of important tags and selectors, and improved the explanations so that the tutorial content actually matched the quiz questions. I also added stronger interactive examples for HTML and CSS instead of relying only on one basic JavaScript demo.

Limitation noticed:
The AI was good at producing an initial structure, but it tended to default to generic educational wording. Without revision, the result would have remained descriptive rather than genuinely explanatory.

Example Prompt Set 2: Quiz Page Logic

A second major use of AI was in the development of the quiz page. This was the most technically demanding part of the project because it involved loading questions dynamically, randomising order, calculating the result, storing attempts, and retrieving reward content from a public API.

Prompt example:
Explain JavaScript concepts needed for a quiz page that loads questions from JSON, randomises the order, validates unanswered questions, calculates score, saves attempts to local storage, and fetches reward content from a public API.

What the AI produced:
The AI explained few useful set of JavaScript concepts with the general structure needed for the quiz. It explained loading data from a JSON file, generating the question interface dynamically, computing the score, and displaying a result. This was helpful as a foundation because it reduced the difficulty of starting from scratch.

Corrections and improvements made:
I revised the logic so that it better matched the project brief. I improved validation for missing answers, strengthened the result display, made the numbering and rendering more consistent, added beforeunload behaviour, stored attempts in local storage, and ensured that reward content was fetched from a public API only when the user passed. I also improved the quiz feedback so users could see more clearly what happened after submission. I mainly used lecture slides and the project brief to get it done.

Limitation noticed:
The AI's first version usually involved the main happy path, but it did not always using the required logics or tech stack, browser behaviour was off, or were missing subtle requirement details. It was a draft, not a finished solution.

What AI Produced Overall

Across the project as a whole, the AI produced several kinds of output. It generated explanations for HTML page structures, CSS styling ideas, JavaScript functions, and refinement advice. It also explained concepts in simpler language, which was helpful when I needed to understand not just what to change, but why that change was needed.

This was valuable because the project was not only about writing code. It was also about making sure the code satisfied the brief, matched the lecture scope, remained explainable in a demo, and fit together as one coherent website.

Critical Evaluation of AI Reliability

The most important lesson from this project was that AI explanations can look correct while still being subtly wrong or incomplete. One example was quiz-related logic that appeared to be right at first glance but did not fully match the wording of the assignment brief. Some early suggestions would have left too much structure fixed in the HTML or would not have communicated enough result detail to the user. These were not obvious major failures, but they mattered because the rubric rewarded precise compliance with the brief, not just code that “kind of works.”

These issues were detected by comparing the logic and concepts against the lecture slides, checking whether the feature behaviour actually matched the rubric, and then testing the result in the browser. In other words, the problem was not always that the AI was completely wrong. The problem was often that it was only partially correct.

This shows why AI should not be treated as a final authority in coding work. Reliability depends on human checking. AI can generate a polished answer quickly, but it cannot be assumed to have fully understood the exact requirements unless the user verifies that carefully.

Risks and Ethical Considerations

There are several risks in using AI assistance during a coding project. One risk is over-reliance. If a student copies AI-generated code without understanding it, they may end up with a submission they cannot explain in a demo. Another risk is accuracy. AI can confidently produce incomplete, misleading, or slightly incorrect suggestions. A third risk is maintainability. Even when generated code works initially, it can become hard to debug later if the person using it does not actually understand how it works.

There is also an academic integrity issue. AI should be used as a support tool, not as a substitute for the student's own learning, judgement, and responsibility. In this project, I used AI to accelerate drafting and clarify ideas, but I still wrote the code myself, reviewed the output, changed it, tested it, and took responsibility for the final decisions. That distinction is ethically important because the aim of the assignment is to demonstrate my own understanding of web development concepts.

Another ethical consideration is attribution. Since AI influenced planning, drafting, revision, and debugging, it should be acknowledged honestly in the project rather than hidden. Responsible use of AI means being transparent about assistance while still ensuring the final work is understood and genuinely owned by the student.

Overall Reflection

Overall, AI was valuable throughout the project, especially in the early planning stage and during debugging. It helped generate ideas, refine wording, and make technical problems easier to approach. However, the project also made it clear that AI is most useful when it is treated as a starting point rather than a final answer.

The biggest lesson I learned is that the quality of AI assistance depends heavily on the quality of human review. The best results came when I questioned the output, checked it against the project brief, tested it in the browser, and adapted it to suit the exact requirements of the assignment. In that sense, AI improved productivity, but it did not replace understanding, responsibility, or critical thinking.
